# AGENTS 工作规范

本文件用于约束在本仓库中工作的所有 agent 行为。除非用户明确覆盖，必须遵循以下规则。

## 1. 运行环境要求

- 代码不在本地系统 Python 直接运行，统一在 Anaconda 虚拟环境 `dl_mnist` 中运行。
- 在提供命令、脚本说明、调试步骤时，默认前提是已激活：
  - `conda activate dl_mnist`

## 2. 依赖安装要求

- 安装依赖时优先使用 `conda` 指令（如 `conda install`）。
- 只有当 `conda` 不可用、或目标包在 `conda` 渠道不可得时，才使用 `pip` 作为补充方案。
- 如同时给出安装方案，必须先给出 `conda` 版本，再给出 `pip` 备选。

## 3. 硬件与算力要求

- 默认目标机器无 NVIDIA GPU，因此实现应优先编写和提供 CPU 可运行版本。
- 若任务本身算力需求明显较高、或性能目标依赖 GPU 才能合理达成，可以设计/提供 GPU 方案。
- 但在完成相关代码后，agent 必须明确提醒用户：
  - 当前任务建议使用 GPU（可租用服务器）运行；
  - 并说明 CPU 下可能出现的耗时或限制。

## 4. 代码与说明输出要求

- 给出的运行命令、训练命令、推理命令应与 `dl_mnist` 环境一致。
- 若代码中包含设备选择逻辑，默认值应为 CPU 友好（例如自动回退到 `cpu`）。
- 当提供 GPU 相关配置时，需同时给出 CPU 兼容路径，避免用户无法在本机验证。

---

如与用户在当前会话中的最新明确指令冲突，以用户最新指令为准。
